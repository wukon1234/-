# 开题报告：商城智能助手系统（基于大语言模型与RAG的电商平台）

## 一、课题背景与研究意义

### 1.1 课题背景
随着电子商务的持续增长与商品SKU规模的爆发式扩张，用户在“检索—对比—决策”链路中面临信息分散、筛选成本高与决策疲劳等问题。传统电商平台主要依赖关键词搜索与分类导航，难以有效覆盖“语义化需求表达”（如“适合学生党、拍照好、续航强的手机”）场景；同时，人工客服响应存在成本高与峰值拥塞等挑战。

近年来，以GPT系列、Claude、通义千问等为代表的大语言模型（LLM）在自然语言理解与生成方面取得突破，使得“对话式检索与推荐”成为提升购物体验的新路径。进一步地，检索增强生成（Retrieval-Augmented Generation，RAG）通过“先检索、后生成”的机制，将实时业务数据与领域知识引入模型上下文，可显著降低模型幻觉并提升答案可解释性与时效性。

### 1.2 研究意义
- **理论意义**：探索LLM + RAG在电商垂直场景中的架构模式与工程落地路径，形成可复用的模块化设计（LLM服务抽象、数据源扩展、检索策略升级等）。
- **实践意义**：在完整电商业务闭环（登录、商品、购物车、订单等）基础上集成智能助手，实现“咨询—推荐—跳转详情—下单”的可演示闭环，提升用户效率与体验。
- **工程教育意义**：覆盖需求分析、系统设计、实现与测试等软件工程全流程，综合运用Spring Boot、Vue3、JWT、SSE等技术，具备典型全栈工程训练价值。

## 二、国内外研究现状与发展趋势（简述）

- **国外**：大模型能力增强推动智能客服与购物助手产品化，RAG在知识密集型问答中成为主流方案；向量检索（Dense Retrieval）与向量数据库（Milvus/FAISS/pgvector等）支撑语义召回。
- **国内**：通义千问、文心一言、混元等国产模型成熟度提升，电商平台持续推进智能化客服与购物助手；在工程落地上强调成本、延迟与合规，常见做法是分阶段从关键词检索升级到向量检索。

## 三、研究目标与拟解决的关键问题

### 3.1 研究目标
构建一个基于Spring Boot与Vue3的前后端分离电商系统，并集成智能助手模块，实现：
- 完整电商业务功能：用户注册登录、商品浏览/搜索、购物车、订单创建/支付/取消、（可扩展）收货地址管理等。
- 智能助手对话：支持多轮对话与会话管理，提供购物咨询与问题解答。
- RAG商品推荐：根据用户问题检索商品信息，生成回答并返回Top-K推荐商品卡片。
- 流式输出体验：采用SSE实现模型回复逐段输出，提升交互实时性。

### 3.2 关键问题
- **业务数据与模型生成融合**：如何将商品数据以结构化/半结构化方式注入Prompt，减少幻觉并提升可解释性。
- **检索策略的可演进**：如何在可运行版本中实现关键词检索闭环，并预留向量检索升级接口（Embedding/向量库）。
- **实时交互与稳定性**：SSE流式输出下的连接管理、超时与异常处理策略。
- **安全与权限**：JWT无状态鉴权、密码加密存储、接口越权防护、敏感配置保护。

## 四、研究内容与系统功能设计

### 4.1 系统总体功能
系统由“电商业务模块”与“智能助手模块”组成：
- **用户模块**：注册、登录、获取用户信息（JWT鉴权、BCrypt加密）。
- **商品模块**：商品列表/详情/搜索（分页、筛选、排序可扩展）。
- **购物车模块**：添加、查询、更新数量、删除、清空（含库存校验）。
- **订单模块**：创建订单（从购物车结算、扣减库存）、支付（模拟）、取消（恢复库存）、订单列表/详情。
- **智能助手模块（核心）**：会话列表、历史消息、删除会话；对话接口；基于RAG的商品检索与推荐；SSE流式输出。

### 4.2 重点研究内容（与项目实现对应）
- **JWT认证流程**：登录生成Token，拦截器校验与用户上下文注入。
- **SSE流式对话**：后端SseEmitter推送分段文本；前端ReadableStream增量渲染。
- **RAG检索与推荐**：
  - 当前阶段：基于关键词/规则从MySQL商品字段（name/description/specs等）检索Top-K。
  - 升级阶段：引入Embedding服务与向量数据库（如Chroma/Milvus/pgvector）实现语义召回与业务过滤重排。
- **会话管理**：会话与消息持久化，支持分页获取会话列表与历史消息。

### 4.3 拟达到的业务闭环与演示脚本（验收视角）
- **闭环A：电商交易闭环**：注册/登录 → 浏览/搜索商品 → 加入购物车 → 创建订单（扣减库存）→ 支付（状态变更）→ 查询订单列表/详情 → 取消订单（恢复库存，状态更新）。
- **闭环B：智能助手闭环**：进入助手页面 → 新建/选择会话 → 发送问题 → SSE流式回复 → 返回推荐商品卡片 → 点击跳转商品详情页 → 继续追问（上下文连续）→ 会话历史可查询/可删除。

## 五、技术路线与实现方案

### 5.1 总体架构
采用前后端分离架构：
- **前端**：Vue 3 + TypeScript（组件化页面：商品、购物车、订单、智能助手）。
- **后端**：Spring Boot（Controller/Service/Mapper分层），MySQL持久化业务数据。
- **智能助手服务**：LLM服务抽象（可切换供应商）、RAG服务（支持多数据源扩展）、SSE输出。

### 5.2 关键技术选型
- **鉴权安全**：JWT + 拦截器；密码BCrypt加密存储。
- **数据访问**：MySQL（配合Mapper/ORM实现CRUD与事务）。
- **实时输出**：SSE（相比WebSocket更轻量，适配单向流式推送）。
- **RAG扩展**：数据源接口抽象（商品/订单/用户等），支持单源/多源检索策略。

### 5.3 技术可行性分析
- **成熟度**：Spring Boot、Vue3、JWT、SSE均为成熟技术栈；RAG工程化已形成通用模式。
- **实现路径清晰**：先实现关键词检索闭环保证可演示，再逐步升级向量检索提升语义召回质量。
- **风险可控**：LLM调用成本与延迟可通过限流、缓存与降级策略缓解；SSE连接通过超时与异常处理保障稳定。

### 5.4 研究方法（更详细）
- **文献研究法**：调研LLM、RAG、向量检索、对话系统工程化与流式推送的最新进展，形成面向电商场景的可落地方案。
- **原型迭代法**：优先完成“关键词检索RAG闭环”，再围绕检索质量、上下文管理与异常处理逐步增强。
- **模块化开发法**：通过接口抽象（如LLM服务、RAG服务、数据源接口）隔离外部依赖，提升系统可维护性与可扩展性。
- **测试验证法**：以接口测试、典型用例回归与基础性能验证支撑方案可行性。

### 5.5 技术路线（分阶段落地）
- **阶段1（可运行闭环）**：关键词/规则检索 → Top-K候选商品 → Prompt组装 → 生成回答 → 返回推荐商品列表。
- **阶段2（质量提升）**：字段权重与过滤（上架/库存/价格区间）；上下文窗口控制（近N轮）；错误码与降级策略完善。
- **阶段3（语义升级，可选）**：Embedding向量化 → 向量数据库召回 → 业务过滤与重排 → Prompt注入与回答生成。

### 5.6 关键技术与实现要点（与项目文档一致）
- **JWT鉴权**：客户端在请求头携带 `Authorization: Bearer <token>`；后端拦截器校验签名与过期时间，并将用户信息写入上下文，供后续业务校验使用。
- **SSE流式输出**：后端通过SSE按chunk推送回复；结束阶段推送推荐商品事件与done事件；前端以可读流逐步拼接渲染。
- **RAG多数据源**：支持注册商品/订单/用户等数据源；可指定单源检索或多源检索聚合结果，为后续复杂问答预留扩展点。

## 七、预期成果与验收指标

### 7.1 预期成果
- 一套可运行的前后端分离电商系统（含数据库脚本与部署说明）。
- 一套可演示的智能助手功能：SSE流式对话 + RAG商品推荐 + 会话管理。

### 7.2 验收指标（示例）
- **功能**：核心业务功能可用；智能助手可返回流式文本与推荐商品卡片；会话可查询/删除。
- **性能**：普通CRUD接口在常规数据量下满足较快响应；智能助手首字响应满足流式体验要求。
- **安全**：鉴权接口必须携带Token；密码加密存储；越权访问被拒绝并返回统一错误码。
- **可维护性**：分层清晰、统一响应与异常处理、配置外部化、文档可复现部署。

## 八、测试计划与代表性用例（补充）

### 8.1 测试范围
- **认证与安全**：注册/登录、Token校验、401处理、越权访问拦截。
- **业务闭环**：购物车合并与库存校验、订单创建扣减库存、支付/取消状态流转与库存恢复。

### 8.2 代表性用例（示例）
- **用例1：登录成功**：输入正确账号密码 → 返回token → 携带token访问受保护接口成功。
- **用例2：创建订单库存不足**：库存=1、购买数量=2 → 创建失败 → 库存不变、购物车不丢失、提示“库存不足”。
- **用例3：SSE流式对话结束事件**：调用流式接口 → 持续收到文本chunk → 结束时收到推荐商品列表事件与done事件。

## 九、风险分析与应对措施（补充）

| 风险点 | 影响 | 应对策略 |
|---|---|---|
| LLM延迟/超时 | 对话中断、体验变差 | 超时控制；降级提示；必要时重试；SSE提升感知速度 |
| 调用成本 | 成本不可控 | 限流与缓存；压缩上下文；控制最大输出 |
| 检索准确性（关键词阶段） | 推荐不相关 | 字段权重与过滤；Top-K调参；后续升级向量检索 |
| 订单一致性 | 核心数据错误 | 事务控制；边界用例覆盖；错误码与回滚策略 |
| SSE部署缓冲 | 流式失效 | 部署关闭buffer；正确Content-Type；合理超时配置 |
