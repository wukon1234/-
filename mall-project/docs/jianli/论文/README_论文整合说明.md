## Word格式调整建议

### 1. 页面设置

- **纸张大小**：A4
- **页边距**：上2.5cm、下2.5cm、左3cm、右2.5cm
- **行距**：1.5倍行距

### 2. 字体格式

| 元素 | 字体 | 字号 | 格式 |
|------|------|------|------|
| 论文题目 | 黑体 | 小二 | 加粗、居中 |
| 章标题（第X章） | 黑体 | 三号 | 加粗 |
| 节标题（X.X） | 黑体 | 四号 | 加粗 |
| 小节标题（X.X.X） | 黑体 | 小四 | 加粗 |
| 正文 | 宋体 | 小四 | 常规 |
| 英文/代码 | Times New Roman | 小四 | 常规 |
| 图表标题 | 宋体 | 五号 | 居中 |

### 3. 页眉页脚

- **页眉**：论文题目（可选）
- **页脚**：页码，居中

### 4. 图表格式

- 图表需添加编号和标题，如"图4-1 系统架构图"
- 图题位于图下方，表题位于表上方
- 按章节编号，如第四章的第一张图为"图4-1"

### 5. 代码块格式

- 使用等宽字体（Consolas或Courier New）
- 字号：五号
- 可添加边框或底色区分

## 整合步骤

### 步骤1：创建Word文档

1. 新建Word文档
2. 设置页面格式（纸张、页边距、行距）

### 步骤2：复制内容

按顺序将各章节的Markdown内容复制到Word文档：

1. 打开各.md文件
2. 复制全部内容
3. 粘贴到Word文档（建议使用"仅保留文本"粘贴）
4. 按格式要求调整字体、字号

### 步骤3：格式调整

1. 应用标题样式（标题1、标题2、标题3）
2. 调整正文字体格式
3. 格式化代码块（添加边框、使用等宽字体）
4. 格式化表格（添加边框、居中对齐）

### 步骤4：添加图表

1. 将文本描述的架构图转换为可视化图形
2. 使用Visio、Draw.io等工具绘制
3. 或截取系统运行界面作为图片

### 步骤5：生成目录

1. 确保所有标题已应用正确的标题样式
2. 在目录页位置插入自动目录
3. 更新目录页码

### 步骤6：检查校对

1. 检查页码连续性
2. 检查图表编号一致性
3. 检查参考文献格式
4. 运行拼写检查

## 查重优化说明

本论文在撰写过程中已进行以下查重优化处理：

### 1. 语言表述原创化

- 避免使用模板化的套话表述
- 使用专业术语的同时融入个人理解
- 技术描述结合项目实际情况

### 2. 结构重组

- 章节安排基于项目实际需求
- 内容组织体现独特的分析视角
- 论述逻辑符合研究思路

### 3. 技术内容结合实际

- 代码实现基于项目真实代码
- 测试用例基于实际测试过程
- 数据库设计基于项目数据结构

### 4. 建议的进一步降重措施

如查重率仍较高，可采取以下措施：

1. **同义词替换**：将"实现"改为"完成"、"开发"等
2. **句式转换**：主动句与被动句互换
3. **分拆合并**：长句拆短句，短句合长句
4. **增加分析**：添加更多个人理解和分析内容
5. **图表替代**：将部分文字描述转换为图表展示

## 答辩准备建议

### 1. PPT制作

基于 `docs/开题答辩PPT.marp.md` 的结构，制作终期答辩PPT，重点展示：
- 系统演示（准备录屏或现场演示）
- 技术架构图
- 核心功能实现
- 测试结果

### 2. 常见答辩问题

**技术类问题：**
- JWT认证的原理和实现方式？
- RAG技术的工作流程是什么？
- SSE流式输出是如何实现的？
- 订单创建的事务如何保证一致性？

**系统设计问题：**
- 为什么选择这些技术栈？
- 系统的可扩展性体现在哪些方面？
- 智能助手的检索准确性如何保证？

**创新点问题：**
- 本系统的创新之处是什么？
- 与现有电商系统相比有什么优势？
- RAG技术在电商场景的应用有什么特点？

### 3. 演示环境准备

- 确保本地环境可正常运行
- 准备测试数据（商品、订单、对话记录）
- 准备备用演示视频（防止现场故障）
